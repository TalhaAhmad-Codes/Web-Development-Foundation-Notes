# Case Sensitivity Rules in HTML

HTML follows **specific case sensitivity rules** that beginners must understand to avoid confusion when moving to CSS, JavaScript, and frameworks.

---

## HTML Tags

* HTML tags are **case-insensitive**
* Recommended practice is **lowercase**

✔ Valid:

```html
<p>Paragraph</p>
<P>Paragraph</P>
```

✔ Best Practice:

```html
<p>Paragraph</p>
```

---

## HTML Attributes

* Attribute names are **case-insensitive**
* Attribute values **may be case-sensitive**

✔ Valid:

```html
<input TYPE="text">
```

✔ Best Practice:

```html
<input type="text">
```

---

## Attribute Values (Important)

Some attribute values **are case-sensitive**, especially when used with:

* CSS
* JavaScript
* File systems
* Frameworks

Example:

```html
<img src="Logo.png">
```

⚠ On Linux-based servers (Azure):

```text
logo.png ≠ Logo.png
```

---

## `id` and `class` Attributes

* Values are **case-sensitive** when used in CSS & JavaScript

```html
<div id="Main"></div>
```

```css
#main { }   ❌
#Main { }   ✔
```

---

## File & Path Names

* URLs and file paths are **case-sensitive** on most servers

```html
<link rel="stylesheet" href="Styles.css">
```

❌ Breaks if file is `styles.css`

---

## JavaScript Interaction

JavaScript is **strictly case-sensitive**.

```js
document.getElementById("Header"); // ❌
document.getElementById("header"); // ✔
```

---

## Best Practices (Recommended)

* Always write HTML tags in **lowercase**
* Keep `id` and `class` values consistent
* Match file names exactly
* Follow a single naming convention

---

## Quick Summary

| Item                  | Case-Sensitive |
| --------------------- | -------------- |
| HTML tags             | ❌              |
| Attribute names       | ❌              |
| Attribute values      | ⚠ Sometimes    |
| `id` / `class` values | ✅              |
| File paths            | ✅              |
| JavaScript            | ✅              |

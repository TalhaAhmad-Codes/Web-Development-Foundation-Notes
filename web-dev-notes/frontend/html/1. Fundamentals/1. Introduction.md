# What is HTML & How the Web Works

---

## What is HTML?

**HTML (HyperText Markup Language)** is the **standard markup language** used to create and structure content on the web.

* It defines **what content is shown** on a web page
* It is **not a programming language** (no logic, loops, or conditions)
* It uses **elements (tags)** to describe content meaning

HTML acts as the **foundation of every website**.

### Key Responsibilities of HTML

* Structure content (headings, paragraphs, lists)
* Define page layout (header, footer, sections)
* Embed media (images, videos, audio)
* Connect with CSS and JavaScript

---

## Why HTML is Important

* Every browser understands HTML
* CSS and JavaScript **depend on HTML**
* Required for:

  * Web Development
  * ASP.NET MVC / Razor Pages
  * Blazor (HTML-based rendering)

## What is HyperText?

**HyperText** means **text that links to other documents**.

Example:

```html
<a href="https://example.com">Go to Example</a>
```

This allows:

* Navigation between pages
* Linking frontend to backend APIs
* Routing in web apps

## How the Web Works (High-Level)

### Step-by-Step Flow

1. User enters a URL in browser
2. Browser sends an **HTTP request**
3. Server processes request
4. Server returns a response (HTML, CSS, JS)
5. Browser renders the page

```
Browser â†’ HTTP Request â†’ Server
Server â†’ HTML Response â†’ Browser â†’ Render Page
```

## Client vs Server (Beginner View)

### Client (Browser)

* Chrome, Edge, Firefox
* Renders HTML
* Executes JavaScript
* Applies CSS

### Server

* ASP.NET Core / Node.js / Java
* Handles logic
* Connects to database
* Returns HTML or JSON

## Static vs Dynamic HTML

### Static HTML

* Same content for every user
* No backend logic

```html
<p>Welcome to my website</p>
```

### Dynamic HTML

* Content changes
* Generated by server or JavaScript

Example:

```html
<p>Welcome, John</p>
```

(Generated using Razor, JS, or API data)

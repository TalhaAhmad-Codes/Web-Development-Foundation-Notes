# HTML vs CSS vs JavaScript

### Roles & Responsibilities in Web Development

---

## Overview

Modern web development is built on **three core technologies**:

| Technology | Purpose      |
| ---------- | ------------ |
| HTML       | Structure    |
| CSS        | Presentation |
| JavaScript | Behavior     |

Each has a **clear responsibility** and should not overlap in purpose.

---

## HTML (Structure)

### What HTML Does

HTML defines the **structure and meaning** of web content.

* Creates elements like headings, paragraphs, buttons
* Organizes content using semantic tags
* Acts as the **skeleton of a web page**

### What HTML Should Handle

* Page layout structure
* Text content
* Images, links, forms
* Accessibility semantics

### Example (HTML Only)

```html
<h1>User Profile</h1>
<p>Name: Talha</p>
<button>Save</button>
```

üß† **Key Point:**
HTML does **not** control colors, animations, or interactions.

---

## CSS (Styling & Layout)

### What CSS Does

CSS (Cascading Style Sheets) controls **how HTML looks**.

* Colors, fonts, spacing
* Page layout (Flexbox, Grid)
* Responsive design
* Animations & transitions

### What CSS Should Handle

* Visual design
* UI layout
* Responsive behavior
* Theme customization

### Example (CSS Styling HTML)

```css
h1 {
    color: darkblue;
}

button {
    padding: 10px;
    background-color: green;
    color: white;
}
```

üß† **Key Point:**
CSS **never creates content** ‚Äî it only styles existing HTML.

---

## JavaScript (Behavior & Logic)

### What JavaScript Does

JavaScript adds **interactivity and dynamic behavior** to web pages.

* Handles user actions
* Modifies HTML & CSS dynamically
* Communicates with APIs
* Implements logic and conditions

### What JavaScript Should Handle

* Button clicks
* Form validation
* API calls
* DOM manipulation

### Example (JavaScript Interaction)

```html
<button onclick="showMessage()">Click Me</button>

<script>
function showMessage() {
    alert("Hello from JavaScript!");
}
</script>
```

üß† **Key Point:**
JavaScript controls **how the page behaves**, not how it looks.

---

## Clear Separation of Concerns

### Good Practice

```text
HTML ‚Üí What is on the page
CSS  ‚Üí How it looks
JS   ‚Üí How it behaves
```

### Bad Practice (Avoid)

```html
<button style="color:red;" onclick="alert('Hi')">
    Click Me
</button>
```

Why ‚ùå?

* Hard to maintain
* Breaks separation of concerns
* Not scalable for large apps

---

## Combined Example (Clean Approach)

### HTML

```html
<button id="saveBtn">Save</button>
```

### CSS

```css
#saveBtn {
    background-color: blue;
    color: white;
}
```

### JavaScript

```html
<script>
document.getElementById("saveBtn")
    .addEventListener("click", () => {
        alert("Saved successfully!");
    });
</script>
```

‚úî Clean
‚úî Maintainable
‚úî Scalable

* HTML is **mandatory**
* CSS improves **visual experience**
* JavaScript enables **interaction**
* All three together create a complete web experience
* Mastering these makes learning **React, Angular, Blazor, ASP.NET** easier

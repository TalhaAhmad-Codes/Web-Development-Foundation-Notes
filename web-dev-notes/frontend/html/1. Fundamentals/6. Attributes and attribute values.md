# HTML Attributes & Attribute Values

HTML **attributes** provide **additional information** about elements and control their **behavior and appearance**.

---

## What is an HTML Attribute?

### Definition

An **HTML attribute** is a **name‚Äìvalue pair** added to an HTML tag to modify or describe an element.

### Basic Syntax

```html
<tagname attribute="value">
```

### Example

```html
<a href="https://example.com">Visit</a>
```

* `href` ‚Üí attribute name
* `"https://example.com"` ‚Üí attribute value

---

## Where Attributes Are Written

* Always written in the **opening tag**
* Never written in closing tags

‚ùå Incorrect

```html
</a href="link">
```

‚úî Correct

```html
<a href="link">
```

---

## Attribute Values

### Definition

**Attribute values** define the **settings or data** of an attribute.

### Rules

* Enclosed in **quotes** (single or double)
* Double quotes are recommended
* Values can be:

  * Text
  * Numbers
  * URLs
  * Keywords

### Example

```html
<img src="logo.png" alt="Company Logo">
```

---

## Commonly Used Attributes

### `id` Attribute

* Uniquely identifies an element
* Must be **unique per page**

```html
<div id="header"></div>
```

Used by:

* CSS
* JavaScript
* Razor / Blazor

---

### `class` Attribute

* Used to group multiple elements
* Can be reused

```html
<p class="error">Invalid input</p>
```

---

### `style` Attribute

* Adds inline CSS

```html
<p style="color:red;">Error message</p>
```

üß† **Note:**
Inline styles are discouraged in real projects.

---

### `title` Attribute

* Displays tooltip on hover

```html
<button title="Click to save">Save</button>
```

---

## Global Attributes

Global attributes can be used on **any HTML element**.

Common examples:

* `id`
* `class`
* `style`
* `title`
* `hidden`

### Example

```html
<p hidden>This content is hidden</p>
```

---

## Boolean Attributes

### Definition

Boolean attributes **do not require a value**.

* Presence = true
* Absence = false

### Example

```html
<input type="checkbox" checked>
```

Other examples:

* `disabled`
* `readonly`
* `required`
* `autofocus`

---

## Multiple Attributes in One Element

```html

    type="text"
    id="username"
    class="input-field"
    placeholder="Enter username"
    required
>
```

‚úî Clean
‚úî Readable
‚úî Maintainable

## Attribute Order (Best Practice)

While browsers don‚Äôt care about order, **readability matters**:

```html

    type="text"
    id="email"
    class="form-control"
    placeholder="Enter email"

```

## Attributes in Real Projects

### In ASP.NET / Razor

```html
<input asp-for="Email" class="form-control" />
```

### In JavaScript

```js
document.getElementById("email").value;
```

### In CSS

```css
input[type="text"] {
    border: 1px solid gray;
}
```

Attributes act as **bridges** between HTML, CSS, and JavaScript.
